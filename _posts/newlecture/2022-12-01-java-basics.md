Open JDK
JDK는 JCP(Java Community Process) 멤버십에 포함된 여러 기업들에 의해서 만들어지는 오픈소스이다.
JCP 멤버십에서 만들어진 초안이 OpenJDK이며 오라클 JDK는 그것을 오라클은 자사의 패키지를 포함한 배포본이다.

환경 변수
환경 변수란?
OS가 시작할 때 읽는 변수다. 환경 변수는 특정 사용자만을 위한 변수(사용자 변수)로 설정하거나 시스템 전체에서 사용할 수 있는 변수(시스템 변수) 두가지가 있다.
환경변수를 사용하는 이유
프로그램을 실행시키려면 명령어를 입력할 때마다 프로그램이 위치한 경로를 같이 입력해주거나 프로그램이 위치한 경로에 이동해서 명령어를 입력해야하는 불편한 과정을 거쳐야한다. 터미널에 명령어를 입력하면 OS는 먼저 내장 명령어를 검색한다. 내부 명령어에서 명령어를 찾을 수 없다면 OS는 다음으로 환경변수 path에 저장된 경로에서 명령어를 검색한다. 즉 내가 자주 사용하는 프로그램들을 환경변수 path에 저장하면 CLI 환경에서 프로그램을 보다 간편하게 사용할 수 있다.

컴파일과 실행
컴파일이란?
소스코드를 기계어로 번역하는 것.
컴파일러
프로그래밍언어를 기계어로 번역하기 위해 사용되는 번역기.
자바 소스파일 컴파일 방법 : javac example.java
컴파일된 코드 실행 방법 : java example
인터프리터로 코드 실행 : java example.java
java는 자바코드의 실행을 돕는 가상머신이다. 인터프리터와 GC등을 내장하고 있다.

프로젝트 파일

 IDE의 설정을 위해 필요한 파일이다. 어떤 IDE를 사용하냐에 따라서 종속적으로 달라진다.


src 폴더
소스파일이 저장되는 폴더다. src는 source의 약자다.
bin 폴더
소스파일이 중간 컴파일 결과인 class 파일이 저장되는 폴더다. bin은 binary의 약자다.

workbench

 코드 개발을 위한 작업대이다. workbench에 코드 개발에 필요한 도구(코드 편집기, 파일 탐색기)등을 올려 놓을 수 있다. workbench에 다양한 plugin을 설치해서 입맛대로 개발 환경(IDE)을 조성할 수도 있다.


perspective

 작업대에 필요한 도구만을 올려 놓는 일이다. 어떤 perspective를 사용하냐에 따라서 eclipse의 메뉴와 화면 구성이 달라진다.


빌드시점

 자동빌드 설정을하면 소스코드의 변경을 저장할 때 컴파일된다.


주의점

 IDE를 거치지않고 workspace에서 직접 파일의 추가/수정/삭제하는 것은 동기화문제가 발생할수 있기때문에 하지 않는것이 좋다. 동기화문제가 발생할때는 프로젝트를 업데이트를 해야한다.
